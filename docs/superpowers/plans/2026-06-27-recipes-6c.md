# `@csm-lab/recipes` operator-lifecycle families (increment 6c) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the operator-lifecycle recipe families to `@csm-lab/recipes` — 15 shared recipes (manager/reward address proposal+confirm ×4, deposit/unvet/exit/slash/withdraw, penalty report/cancel/settle/compensate ×4, bond ops ×2) — all mechanical on the proven 6b `actAs` engine.

**Architecture:** Each recipe is a thin `read → actAs(privileged) → write → return` transcription of a `script/fork-helpers/NodeOperators.s.sol` function, dropping its post-condition `assertEq`s (recipes return what they did; viem reverts throw naturally). All are **shared** (csm + cm) and call the module via `contract(ctx, 'module')` (= `csModuleAbi`) or `contract(ctx, 'Accounting')`. The privileged sender for each comes straight from the Solidity `broadcast*` modifier it carried.

**Tech Stack:** TypeScript (ESM, `moduleResolution: Bundler`), viem `^2.53`, `@csm-lab/receipts` (vendored ABIs), the existing 6b primitives (`connect`/`actAs`/`roleMember`/`contract`). No Foundry. Vitest + the existing hermetic fake-client harness.

## Global Constraints

These bind every task (same package conventions as 6b — see `docs/superpowers/plans/2026-06-27-recipes-6b.md`).

- **No Foundry.** anvil is the target chain over RPC via viem only.
- **Recipes prepare state; they do NOT assert.** Drop the Solidity `assertEq`s. Each recipe **returns what it did** (a computed value, or `void` for a pure state change) and **throws on no-op / bad input** (a revert throws via viem). Do NOT re-read post-state just to assert it.
- **All recipes are shared** (csm + cm). Resolve the module via `contract(ctx, 'module')` (returns `csModuleAbi` for both modules — only the address switches by `ctx.module`; the called functions are all on the shared IBaseModule surface). Bond ops use `contract(ctx, 'Accounting')`.
- **Impersonation via `actAs(ctx, who, fn)`** (6b): funds + impersonates `who`, runs `fn(from)`, always stops. The target `who` per recipe comes from its Solidity `broadcast*` modifier (table below).
- **viem writes pass `account: from` + `chain: null`** (the client has no chain). To capture a state-changing call's RETURN value, use the simulate→write pattern: `const { result, request } = await client.simulateContract({...}); await client.writeContract({ ...request, chain: null })`.
- **ABIs vendored** via `@csm-lab/receipts` (`csModuleAbi`, `accountingAbi`). Never import ABIs from the SDK.
- **Toolchain:** ESM extensionless imports (`from './x'`, never `'./x.js'`); `import type` for type-only imports; `noUncheckedIndexedAccess` is ON (guard array access); no DOM lib; prefer `Array#toSorted()`. New exports are **APPENDED** to `src/index.ts` (never rewrite prior exports). `tsconfig.json` already includes `src/**/*` + `test/**/*` so `pnpm types` typechecks tests.
- **Reads are already abitype-narrowed** (`contract(ctx,'module')` returns the concrete `csModuleAbi`, not a union) — access struct fields directly (`op.managerAddress`); do NOT widen to `Abi`/`any`. Add a minimal cast only if `tsc` actually errors.
- **Tests are hermetic** (CLAUDE.md): inject the existing `makeFakeClient` (its `reads` map is keyed by `functionName`; `simulate` supplies `{ result, request }`); assert calls, encodings, and `impersonate → send → stop` order; resolve impersonation targets from `fakeCtx` fixtures. No network, no chain. No fork tests added here (the 6b smoke already covers the live path).
- **Per-package gates before "done":** `build` · `types` · `test` · `oxlint tools/recipes` · `prettier --check "tools/recipes/**/*.{ts,json}"`.

### Source facts (contracts repo HEAD `b8efc56`) — the 15 recipes

All module functions are on **`csModuleAbi`** (IBaseModule, shared by CSModule + CuratedModule). Bond functions are on **`accountingAbi`** (IAccounting). `getNodeOperator(noId)` returns the NodeOperator struct (fields incl. `managerAddress`, `proposedManagerAddress`, `proposedRewardAddress` — all `address`).

| Recipe | Module/Accounting call (exact) | Impersonate (`broadcast*` → address) |
| --- | --- | --- |
| `proposeManager` | `proposeNodeOperatorManagerAddressChange(uint256 noId, address proposed)` | `getNodeOperator(noId).managerAddress` |
| `confirmManager` | `confirmNodeOperatorManagerAddressChange(uint256 noId)` | `getNodeOperator(noId).proposedManagerAddress` |
| `proposeReward` | `proposeNodeOperatorRewardAddressChange(uint256 noId, address proposed)` | `getNodeOperator(noId).managerAddress` ⚠️ (manager, not reward — confirmed) |
| `confirmReward` | `confirmNodeOperatorRewardAddressChange(uint256 noId)` | `getNodeOperator(noId).proposedRewardAddress` |
| `unvet` | `decreaseVettedSigningKeysCount(bytes nodeOperatorIds, bytes vettedSigningKeysCounts)` | `ctx.addresses.stakingRouter` |
| `exit` | `updateExitedValidatorsCount(bytes nodeOperatorIds, bytes exitedValidatorsCounts)` | `ctx.addresses.stakingRouter` |
| `deposit` | `batchDepositInfoUpdate(uint256)`, `getStakingModuleSummary()→(uint256,uint256,uint256)`, `obtainDepositData(uint256 count, bytes "")→(bytes pubkeys, bytes sigs)` | `ctx.addresses.stakingRouter` |
| `slash` | `reportValidatorSlashing(uint256 noId, uint256 keyIndex)` | `ctx.addresses.Verifier` |
| `withdraw` | `reportRegularWithdrawnValidators(WithdrawnValidatorInfo[] infos)` | `ctx.addresses.Verifier` |
| `reportPenalty` | `reportGeneralDelayedPenalty(uint256 noId, bytes32 penaltyType, uint256 amount, string details)` | `roleMember(REPORT_GENERAL_DELAYED_PENALTY_ROLE)` |
| `cancelPenalty` | `cancelGeneralDelayedPenalty(uint256 noId, uint256 amount)` | `roleMember(REPORT_GENERAL_DELAYED_PENALTY_ROLE)` |
| `settlePenalty` | `settleGeneralDelayedPenalty(uint256[] noIds, uint256[] maxAmounts)` | `roleMember(SETTLE_GENERAL_DELAYED_PENALTY_ROLE)` |
| `compensatePenalty` | `compensateGeneralDelayedPenalty(uint256 noId)` | `getNodeOperator(noId).managerAddress` ⚠️ (NatSpec requires manager; fork helper's `stranger` is wrong) |
| `addBond` | `depositETH(uint256 noId)` **payable**, `msg.value = amount` (the permissionless 1-arg overload) | `getNodeOperator(noId).managerAddress` |
| `createBondDebt` | `penalize(uint256 noId, uint256 amount) → (bool penaltyCovered)` | `ctx.addresses` module address (impersonate the module to call Accounting) |

- **`WithdrawnValidatorInfo` struct:** `{ uint256 nodeOperatorId; uint256 keyIndex; uint256 exitBalance; uint256 slashingPenalty; bool isSlashed }`. The fork helper sets `isSlashed = slashingPenalty > 0`.
- **Byte packing (unvet/exit):** `nodeOperatorIds` = `bytes8(uint64(noId))` per op; counts = `bytes16(uint128(count))` per op. viem: `toHex(noId, { size: 8 })` / `toHex(count, { size: 16 })`. Single-op recipes pass one packed value.
- **deposit count:** derive from `obtainDepositData`'s returned `pubkeys` (length / 48) via simulate→write — NOT a before/after summary diff (avoids needing sequential fake reads). Cap `count` to `getStakingModuleSummary()`'s 3rd field (`depositableValidatorsCount`); throw if `requested > 0 && capped == 0` (the silent-no-op hazard).
- **Role constants:** `REPORT_GENERAL_DELAYED_PENALTY_ROLE = keccak256("REPORT_GENERAL_DELAYED_PENALTY_ROLE")`, `SETTLE_GENERAL_DELAYED_PENALTY_ROLE = keccak256("SETTLE_GENERAL_DELAYED_PENALTY_ROLE")`.
- **penaltyType default** (`reportPenalty`): the fork helper used `bytes32(abi.encode(1))` = `toHex(1n, { size: 32 })`. **details default:** `"fork-penalty"`.

`exitRequest` (VEBO 2-step), `targetLimit`/forced/off, `activateKeys`/`reportBalance`/`increaseAllocatedBalance`/`topUpActiveKeys`, the typed reads beyond `operatorInfo`, `removeKey` — all later increments (6d/6f). `clActivate` (6d), rewards (6e), CLI (6g).

```ts
import type { Hex } from '@csm-lab/receipts';
import { actAs } from '../act-as';
import { contract, type Ctx } from '../context';

/** Propose a new manager address for an operator (signed by the current manager). */
export async function proposeManager(ctx: Ctx, opts: { noId: bigint; proposed: Hex }): Promise<void> {
  const m = contract(ctx, 'module');
  const op = await ctx.client.readContract({ ...m, functionName: 'getNodeOperator', args: [opts.noId] });
  await actAs(ctx, op.managerAddress, (from) =>
    ctx.client.writeContract({
      ...m,
      functionName: 'proposeNodeOperatorManagerAddressChange',
      args: [opts.noId, opts.proposed],
      account: from,
      chain: null,
    }),
  );
}

/** Confirm the proposed manager address (signed by the proposed manager). */
export async function confirmManager(ctx: Ctx, opts: { noId: bigint }): Promise<void> {
  const m = contract(ctx, 'module');
  const op = await ctx.client.readContract({ ...m, functionName: 'getNodeOperator', args: [opts.noId] });
  await actAs(ctx, op.proposedManagerAddress, (from) =>
    ctx.client.writeContract({
      ...m,
      functionName: 'confirmNodeOperatorManagerAddressChange',
      args: [opts.noId],
      account: from,
      chain: null,
    }),
  );
}

/** Propose a new reward address (signed by the MANAGER, per the on-chain access rule). */
export async function proposeReward(ctx: Ctx, opts: { noId: bigint; proposed: Hex }): Promise<void> {
  const m = contract(ctx, 'module');
  const op = await ctx.client.readContract({ ...m, functionName: 'getNodeOperator', args: [opts.noId] });

      ...m,
      functionName: 'proposeNodeOperatorRewardAddressChange',
      args: [opts.noId, opts.proposed],

      chain: null,
    }),
  );
}

/** Confirm the proposed reward address (signed by the proposed reward address). */
export async function confirmReward(ctx: Ctx, opts: { noId: bigint }): Promise<void> {
  const m = contract(ctx, 'module');
  const op = await ctx.client.readContract({ ...m, functionName: 'getNodeOperator', args: [opts.noId] });

}
```

- [ ] **Step 1: Write the failing test** — `tools/recipes/test/deposit.test.ts`

```ts
import { describe, expect, it } from 'vitest';
import { deposit } from '../src/recipes/deposit';
import { makeFakeClient } from './helpers/fake-client';
import { A, fakeCtx } from './helpers/book';

// 2 pubkeys = 96 bytes = 192 hex chars after 0x
const TWO_PUBKEYS = `0x${'ab'.repeat(48 * 2)}` as const;

describe('deposit', () => {
  it('flushes, caps to depositable, deposits, and returns the count from returned pubkeys', async () => {
    const fc = makeFakeClient({
      reads: { getStakingModuleSummary: [0n, 10n, 5n] }, // depositable = 5
      simulate: { result: [TWO_PUBKEYS, '0x'], request: { __dep: true } },
    });
    const ctx = fakeCtx('csm', fc.client, { CSModule: A(0x01) });

    const res = await deposit(ctx, { count: 2 });
    expect(res.deposited).toBe(2n);

    // flush write happened
    const writes = fc.byMethod('writeContract') as any[];
    expect(writes.some((w) => w.functionName === 'batchDepositInfoUpdate')).toBe(true);
    // obtainDepositData simulated with the (capped) count + empty calldata, as the staking router
    const sim = fc.byMethod('simulateContract')[0] as any;
    expect(sim.functionName).toBe('obtainDepositData');
    expect(sim.args).toEqual([2n, '0x']);
    expect(sim.account).toBe(ctx.addresses.stakingRouter);
    // the create write reused the simulate request
    expect(writes.some((w) => w.__dep === true)).toBe(true);
    // impersonated the staking router
    expect(fc.byMethod('impersonateAccount')).toContainEqual({ address: ctx.addresses.stakingRouter });
  });

  it('caps the requested count to depositableValidatorsCount', async () => {
    const fc = makeFakeClient({
      reads: { getStakingModuleSummary: [0n, 10n, 1n] }, // depositable = 1
      simulate: { result: [`0x${'cd'.repeat(48)}`, '0x'], request: { __dep: true } }, // 1 pubkey
    });
    const ctx = fakeCtx('csm', fc.client, { CSModule: A(0x01) });
    const res = await deposit(ctx, { count: 9 });
    const sim = fc.byMethod('simulateContract')[0] as any;
    expect(sim.args).toEqual([1n, '0x']); // capped to 1
    expect(res.deposited).toBe(1n);
  });

  it('throws when a positive request has nothing depositable (silent no-op guard)', async () => {
```

```ts
import { maxUint256, size } from 'viem';
import { actAs } from '../act-as';
import { contract, type Ctx } from '../context';

/**
 * Deposit up to `count` of an operator's depositable keys (StakingRouter-gated). Flushes pending
 * deposit-info, caps to the module's depositable count, and returns the number actually deposited
 * (derived from the keys `obtainDepositData` hands back). Throws if a positive request finds nothing
 * depositable — the Solidity helper silently no-ops there. (Port of `NodeOperators.deposit`.)
 */
export async function deposit(ctx: Ctx, opts: { count: number | bigint }): Promise<{ deposited: bigint }> {
  const m = contract(ctx, 'module');
  const requested = BigInt(opts.count);

  return actAs(ctx, ctx.addresses.stakingRouter, async (from) => {
    await ctx.client.writeContract({
      ...m,
      functionName: 'batchDepositInfoUpdate',
      args: [maxUint256],
      account: from,
      chain: null,
    });

    const summary = await ctx.client.readContract({ ...m, functionName: 'getStakingModuleSummary' });
    const depositable = summary[2]; // (totalExited, totalDeposited, depositableValidatorsCount)
    const capped = requested < depositable ? requested : depositable;
    if (requested > 0n && capped === 0n) {
      throw new Error('@csm-lab/recipes: deposit found nothing depositable for this module');
    }

    const { result, request } = await ctx.client.simulateContract({
      ...m,
      functionName: 'obtainDepositData',
      args: [capped, '0x'],
      account: from,
    });
    await ctx.client.writeContract({ ...request, chain: null });

    const pubkeys = result[0]; // (publicKeys, signatures); 48 bytes per deposited key
    return { deposited: BigInt(size(pubkeys) / 48) };
  });
}
```

> `summary[2]` and `result[0]` are tuple indices into abitype-narrowed return types — both are `bigint`/`Hex` respectively with no cast. If `noUncheckedIndexedAccess` flags `summary[2]`/`result[0]` as possibly-undefined, destructure instead: `const [, , depositable] = summary;` and `const [pubkeys] = result;` (tuple destructuring is not flagged by the rule).

```ts
  ctx: Ctx,
```

```ts
import { actAs } from '../act-as';
import { contract, type Ctx } from '../context';

/** Top up an operator's bond with ETH (permissionless `depositETH`), signed by the manager. */
export async function addBond(ctx: Ctx, opts: { noId: bigint; amount: bigint }): Promise<void> {
  const m = contract(ctx, 'module');
  const acc = contract(ctx, 'Accounting');
  const op = await ctx.client.readContract({ ...m, functionName: 'getNodeOperator', args: [opts.noId] });
  await actAs(ctx, op.managerAddress, (from) =>
    ctx.client.writeContract({
      ...acc,
      functionName: 'depositETH',
      args: [opts.noId],
      account: from,
      value: opts.amount,
      chain: null,
    }),
  );
}

/**
 * Charge a penalty against an operator's bond (creating bond debt if uncovered). `Accounting.penalize`
 * is module-only, so we impersonate the module address. Returns whether the penalty was fully covered.
 */
export async function createBondDebt(
  ctx: Ctx,
  opts: { noId: bigint; amount: bigint },
): Promise<{ penaltyCovered: boolean }> {
  const moduleAddress = contract(ctx, 'module').address;
  const acc = contract(ctx, 'Accounting');
  return actAs(ctx, moduleAddress, async (from) => {
    const { result, request } = await ctx.client.simulateContract({
      ...acc,
      functionName: 'penalize',
      args: [opts.noId, opts.amount],
      account: from,
    });
    await ctx.client.writeContract({ ...request, chain: null });
    return { penaltyCovered: result };
  });
}
```

> **`depositETH` is overloaded** in the Accounting ABI: `depositETH(uint256 noId)` (permissionless, used here) and `depositETH(address from, uint256 noId)` (module-only). viem resolves the overload by arg count — `args: [opts.noId]` (one arg) targets the 1-arg permissionless version. If viem/abitype raises an overload-ambiguity TS error on `functionName: 'depositETH'`, report it (do NOT silence with `any`); the single-arg call is unambiguous at the value level.
