# Phase 1 — Rename to `sm-lab` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename the monorepo from `csm-lab`/`@csm-lab` to `sm-lab`/`@sm-lab`, drop the `-mock` suffix from the two app packages/dirs, and rename all bins to `sm-*` — ending with a fully green build under the new names.

**Architecture:** A mechanical, surgical codemod applied in ordered passes, each verified by a grep invariant and/or the build. This is *not* TDD (no new behavior) — the "tests" are (a) grep assertions that a rename is complete and (b) the existing suite + build staying green. Passes are ordered so each leaves the tree installable/buildable.

**Tech Stack:** pnpm workspaces + catalog, tsdown, vitest, oxlint, prettier, turbo, Docker.

## Global Constraints

- Repo `sm-lab`; npm scope `@sm-lab`; 8 packages.
- App package renames (drop `-mock`): `@csm-lab/cl-mock` → `@sm-lab/cl`, `@csm-lab/ipfs-mock` → `@sm-lab/ipfs`. Dirs: `apps/cl-mock` → `apps/cl`, `apps/ipfs-mock` → `apps/ipfs`.
- Other packages: scope only — `@sm-lab/{keys,merkle,recipes,core,config,receipts}` (dirs unchanged).
- Bins: `csm-cl-mock`→`sm-cl`, `csm-ipfs-mock`→`sm-ipfs`, `csm-merkle`→`sm-merkle`, `csm-keys`→`sm-keys`, `csm-recipes`→`sm-recipes`.
- **DO NOT rename** (out of scope for Phase 1, all stay verbatim): env vars `CL_MOCK_URL`/`IPFS_MOCK_URL`/etc., variable names (`clMockUrl`), the filename `recipes/src/cl-mock.ts`, and prose describing "the mock". Only package-name / dir / docker-service / bin drop `-mock`.
- **DO NOT sweep** with the scope codemod: `pnpm-lock.yaml` (regenerated by install), historical records under `docs/superpowers/plans/` and `docs/superpowers/specs/` (point-in-time; keep old name), and `docs/migration.md` (deleted in Phase 2).
- Node engine stays `>=20`, deps unchanged — node-24 bump is Phase 2, not here.
- ESM extensionless imports; `moduleResolution: Bundler`; package `tsconfig` uses *relative* extends (unaffected by scope — only string mentions change).
- Per-package done-gate: `build` · `types` · `test` · `oxlint <dir>` · `prettier --check`.
- Nothing is published to npm yet; the ~21 pending `.changeset/*.md` MUST be rewritten so the coordinated first publish ships correct names.
